GitHub to GitLab Auto Update

Questo progetto permette di configurare automaticamente il mirroring dei repository dal tuo account GitHub a GitLab, rendendo semplice mantenere sincronizzati tutti i tuoi repository.

Informazioni Generali

  • Account GitHub: bocaletto-luca
  • Account GitLab: bocaletto-luca
  • Nome del Progetto: github-to-gitlab-auto-update
  • Descrizione: Script Python per automatizzare la sincronizzazione (mirroring) dei repository GitHub in GitLab.

Requisiti

  • Python 3
  • Modulo Python: requests (installabile con pip install requests)
  • Accesso alle API di GitHub e GitLab

Configurazione

1. Ottenere i Token di Accesso Personale

  1. Token GitHub:

    Vai su GitHub Settings > Tokens e genera un nuovo token con il permesso repo (e altri scope se necessari).

  2. Token GitLab:

    Accedi a GitLab Personal Access Tokens e crea un nuovo token con lo scope api.

  3. Importante: Non inserire mai i token in chiaro nel repository pubblico. Modifica il file dello script in modo da leggere i token da variabili d'ambiente o file di configurazione locali.

2. Configurare lo Script

Modifica il file github_to_gitlab_auto_update.py e aggiorna le seguenti variabili: 

GITHUB_USERNAME = "bocaletto-luca"  # Il tuo username GitHub
GITHUB_TOKEN  = "your_github_token_here"  # Sostituisci con il tuo token GitHub
GITLAB_TOKEN  = "your_gitlab_token_here"  # Sostituisci con il tuo token GitLab

3. Eseguire lo Script

Dal terminale, rendi eseguibile lo script (se necessario) e poi eseguilo: 

chmod +x github_to_gitlab_auto_update.py
python3 github_to_gitlab_auto_update.py

Come Funziona

Lo script esegue le seguenti operazioni:

  • Recupera tutti i progetti GitLab in cui sei membro tramite l'API di GitLab.
  • Per ciascun progetto, costruisce l'URL del corrispondente repository su GitHub (basato sul nome della repository).
  • Configura il mirroring nel progetto GitLab verso il repository su GitHub, in modo che ogni aggiornamento effettuato su GitHub venga propagato automaticamente su GitLab.

Note Importanti

  • Verifica che il nome del repository su GitHub corrisponda esattamente al campo path del progetto su GitLab.
  • Testa lo script su un sottoinsieme di repository prima di applicarlo a tutti i tuoi progetti.
  • Consulta i log di esecuzione per eventuali errori e assicurati che i token siano stati configurati correttamente.
